Practice Quiz: Code Reviews




Q: When should we respond to comments from collaborators and reviewers?

* Always




Q: What is a nit?

* A trivial comment or suggestion




Q: Select common code issues that might be addressed in a code review. (Check all that apply)

* Using unclear names
* Forgetting to handle a specific condition
* Forgetting to add tests




Q: If we've pushed a new version since we've made a recent change, what might our comment be flagged as?

* Outdated




Q; What are the goals of code review? (Check all that apply)

* Make sure that the contents are easy to understand
* Ensure consistent style
* Ensure we don't forget any important cases
